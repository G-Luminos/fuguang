🚀 Supabase 配置助手

为浮光舰长礼物系统配置数据库和存储

1 登录 Supabase 控制台

点击下方按钮打开 Supabase 项目控制台

打开 Supabase 控制台
2 创建 gift_images 数据表

1. 在左侧菜单点击 SQL Editor

2. 点击 New query

3. 复制下方 SQL 代码并粘贴

4. 点击 Run 执行

-- 创建礼物图片表
CREATE TABLE IF NOT EXISTS gift_images (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  month_id TEXT NOT NULL,
  storage_path TEXT NOT NULL,
  public_url TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_gift_images_month ON gift_images(month_id);
CREATE INDEX IF NOT EXISTS idx_gift_images_sort ON gift_images(sort_order);

-- 启用 RLS
ALTER TABLE gift_images ENABLE ROW LEVEL SECURITY;

-- 策略：允许匿名查看
CREATE POLICY "Anyone can view gift images"
  ON gift_images FOR SELECT
  USING (true);

-- 策略：管理员可以增删改
CREATE POLICY "Admins can manage gift images"
  ON gift_images FOR ALL
  USING (true)
  WITH CHECK (true);
3 创建 Storage Bucket

1. 在左侧菜单点击 Storage

2. 点击 New bucket

3. 填写信息：

  • Name: gifts
  • Public bucket: ✅ 必须勾选

4. 点击 Create bucket

打开 Storage 页面
4 设置 Storage 权限

1. 在 Storage 页面点击 Policies

2. 选择 gifts bucket

3. 添加以下策略：

SELECT 策略（允许所有人查看）：

  • Policy name: Allow public read
  • Allowed operation: SELECT
  • Target roles: anon, authenticated
  • Policy definition: true

INSERT 策略（允许上传）：

  • Policy name: Allow insert
  • Allowed operation: INSERT
  • Target roles: authenticated
  • Policy definition: true

DELETE 策略（允许删除）：

  • Policy name: Allow delete
  • Allowed operation: DELETE
  • Target roles: authenticated
  • Policy definition: true
🎉

配置完成！
